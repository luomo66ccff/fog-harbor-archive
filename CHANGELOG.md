# Changelog

本项目的主要版本变化记录在此文件中。

## [2.1.0] - 2026-07-14

### Added

- 五幕叙事深化：没有下雨的死亡、被保存的声音、照片中的第三方、责任分层，以及仍在同步读取档案的外部节点
- 第二道人影的非绑定临时推理、工具箱/声纹触发的修正流程与本地推理历史
- 可选叙事彩蛋、二周目批注与调查员索引；这些内容不影响主线进度或结局判定
- 桌面端、`390×844` 移动端、二周目与彩蛋隔离性的 Playwright 端到端测试
- 基于 Node.js 22 和 Chromium 的 GitHub Actions 持续集成，以及失败报告与测试产物上传
- 运行时图片和原始设计资源的一致性、完整性与 SHA-256 回归测试

### Changed

- 重写许晚澄、顾惟安、陈牧、唐芷与林知夏的材料，使误导、胁迫协助、保护性隐瞒和调查性改写分阶段回收
- 责任链允许任意 Token 试放，提交时分别统计类型错误、逻辑错误、证据不足与附件矛盾；失败两次或辅助模式下才显示分类
- 普通照片逐区扫描只移动并高亮范围，辅助调查模式才自动确认异常
- 版本号更新为 `2.1.0`，统一单元测试、端到端测试和全量验证脚本
- 原始 PNG 迁移到 `design-assets/source/`，公开运行目录只保留 WebP 与运行时清单
- Docker 构建上下文排除原始设计资源，缩小上传与构建体积

### Fixed

- 照片检查完成计时器现在可在卸载时取消，并保证完成回调只触发一次
- 清理谜题组件中的重复全局 CSS 导入，固定全局样式顺序
- 会话内七次印章与档案藏头进度改为可清理状态，避免清空存档后残留
- 放大 2.1 新增叙事和彩蛋正文及触控目标，避免移动端环境互动遮挡主线窗口

### Compatibility

- 继续使用 `fog-harbor-save-v1` 存档键，保留既有调查流程、正式结局与旧存档兼容策略
