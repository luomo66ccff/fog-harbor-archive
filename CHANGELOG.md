# Changelog

本项目的主要版本变化记录在此文件中。

## [2.2.0] - 2026-07-15

### Added

- 九个一次性导演演出，覆盖时间恢复、纸带锁定、照片复原、推理修正、责任链、外部读取者和三个既有结局；全部可跳过并提供 reduced-motion 静态替代
- 第二轮环境记忆、远处轮廓与六种固定种子的低频异常事件；单轮最多出现两次且不修改任何证据或谜题状态
- 按 RUN 分组的调查记录册、玩家笔记/系统记录分区，以及经过文件名清洗与敏感元数据移除的纯文本导出
- 三个结局各自的短余波与可展开本轮调查回顾
- 桌面端、移动端和调查员索引的固定尺寸 WebP 截图脚本与正式 README 图片
- 每周 Dependabot 与 `docs/security/dependency-audit.md` 依赖公告记录

### Changed

- 归档印章不再显示数字进度，改用墨迹、划痕、纸张位移与最终隐藏文字逐级反馈
- 镜像地图入口改为地图右下角的异常港务符号，首次聚焦只给出印刷方向提示，发现后才显示明确翻转操作
- 谜题解锁通知改为在对应短演出结束后出现；录音演出会短暂压低雨声，责任链按五段依次闭合
- 第二轮调查室降低亮度并显示任务记忆，但不会跳过谜题、自动填答案或改变结局门槛
- 主背景预加载；移动端减少雨雾层；页面隐藏时暂停环境和演出动画；Canvas DPR 继续限制为 2
- 社交封面改为 WebP，公开目录保持零 PNG；原始封面移动至设计资源目录
- Next.js 升级至 `16.2.10`，Cloudflare/Vite 工具链升级至当前兼容版本

### Fixed

- 将 `investigator-file-created` 迁移为 `investigator-index-written`，并由 Seventh 结局页首次实际渲染统一写入，消除双重来源与重复记录
- 通用空闲检测补齐 pointermove、wheel、touchmove、可见性和焦点处理，并对 pointermove 使用 500ms 节流
- 外部读取者演出改为最终卷宗实际挂载后触发；后台标签页会暂停演出与环境事件，不再静默消耗一次性内容
- 调查日志改为记录本轮新增叙事与隐藏发现，并加固 Windows 设备保留名、压缩 IPv6、异常时间戳和被篡改标签的清洗
- 环境声 duck 在设置切换时取消旧自动化，关闭环境声后不会被延迟恢复
- 环境事件计划在同一轮保持固定，达到两次上限后不再尝试第三个事件
- README 截图占位、社交分享图片格式与 CI 失败产物路径完成修复

### Compatibility

- 继续使用 `fog-harbor-save-v1` 与 `fog-harbor-easter-session-v1`
- 新增演出、环境事件、运行记忆、轮次日志与结束时间字段均经过清洗；旧事件 ID 和缺失字段自动迁移
- 保持 `truth`、`trade`、`seventh` 三个结局 ID、主线事实与移动端完整玩法不变

## [2.1.0] - 2026-07-14

### Added

- 五幕叙事深化：没有下雨的死亡、被保存的声音、照片中的第三方、责任分层，以及仍在同步读取档案的外部节点
- 第二道人影的非绑定临时推理、工具箱/声纹触发的修正流程与本地推理历史
- 可选叙事彩蛋、二周目批注与调查员索引；这些内容不影响主线进度或结局判定
- 桌面端、`390×844` 移动端、二周目与彩蛋隔离性的 Playwright 端到端测试
- 基于 Node.js 22 和 Chromium 的 GitHub Actions 持续集成，以及失败报告与测试产物上传
- 运行时图片和原始设计资源的一致性、完整性与 SHA-256 回归测试

### Changed

- 重写许晚澄、顾惟安、陈牧、唐芷与林知夏的材料，使误导、胁迫协助、保护性隐瞒和调查性改写分阶段回收
- 责任链允许任意 Token 试放，提交时分别统计类型错误、逻辑错误、证据不足与附件矛盾；失败两次或辅助模式下才显示分类
- 普通照片逐区扫描只移动并高亮范围，辅助调查模式才自动确认异常
- 版本号更新为 `2.1.0`，统一单元测试、端到端测试和全量验证脚本
- 原始 PNG 迁移到 `design-assets/source/`，公开运行目录只保留 WebP 与运行时清单
- Docker 构建上下文排除原始设计资源，缩小上传与构建体积

### Fixed

- 照片检查完成计时器现在可在卸载时取消，并保证完成回调只触发一次
- 清理谜题组件中的重复全局 CSS 导入，固定全局样式顺序
- 会话内七次印章与档案藏头进度改为可清理状态，避免清空存档后残留
- 放大 2.1 新增叙事和彩蛋正文及触控目标，避免移动端环境互动遮挡主线窗口

### Compatibility

- 继续使用 `fog-harbor-save-v1` 存档键，保留既有调查流程、正式结局与旧存档兼容策略
